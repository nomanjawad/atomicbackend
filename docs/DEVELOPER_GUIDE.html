Skytech Solution Admin Dashboard

A comprehensive admin dashboard built with React and Express.js, powered by Supabase. This guide will help you understand, develop, and extend the application.

React 18 Express.js Supabase PostgreSQL Bootstrap 5

Introduction

Welcome to the Skytech Solution Admin Dashboard developer guide! This document is designed to help new developers understand the codebase and start contributing quickly.

?
What is Skytech Solution Admin?

Skytech Solution Admin is a full-featured content management system (CMS) and admin dashboard that allows administrators to manage:

  • Website Pages - Home, About, Contact, Services, Industries, etc.
  • Blog Posts - Create, edit, publish, and archive articles
  • Media Gallery - Upload and manage images
  • User Management - Create users, assign roles, manage permissions
  • Analytics - Track visitors and page views
  • SEO Scripts - Inject custom scripts for tracking and optimization

Installation

Follow these steps to set up the project on your local machine:

Clone the Repository

Open your terminal and run:

git clone <repository-url>
cd amco-admin
Install Frontend Dependencies

In the project root directory:

npm install

This installs all React and frontend packages defined in package.json.

Install Backend Dependencies

Navigate to the backend folder and install:

cd backend
npm install
cd ..
Configure Environment Variables

Create environment files for both frontend and backend:

Frontend (.env in root folder):

# Create .env file in project root
REACT_APP_API_URL=http://localhost:5000/api

Backend (backend/.env):

# Copy from .env.example and fill in your Supabase credentials
cp backend/.env.example backend/.env

# Then edit backend/.env with your values:
SUPABASE_URL=your_supabase_project_url
SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key
PORT=5000
NODE_ENV=development
CORS_ORIGINS=http://localhost:3000
Set Up Supabase Database

Create a Supabase project and run the migration:

  1. Go to supabase.com and create a new project
  2. Go to the SQL Editor in your Supabase dashboard
  3. Copy the contents of backend/migrations/000_complete_database_schema.sql
  4. Run the SQL to create all tables and policies
Start the Application

You need two terminal windows:

Terminal 1 - Start Backend:

cd backend
npm run dev

Backend runs on http://localhost:5000

Terminal 2 - Start Frontend:

npm start

Frontend runs on http://localhost:3000

Success!
If everything is configured correctly, you should see the login page at http://localhost:3000/sign-in

Understanding React Components

React applications are built from components - reusable pieces of UI. Here's a simple example:

// A simple React component
import React, { useState } from 'react';

function WelcomeCard({ name }) {
    // useState creates a piece of state that can change
    const [count, setCount] = useState(0);

    return (
        <div className="card">
            <h3>Welcome, {name}!</h3>
            <p>You clicked {count} times</p>
            <button onClick={() => setCount(count + 1)}>
                Click me
            </button>
        </div>
    );
}

export default WelcomeCard;
Key Concepts

Request Flow Example

// 1. Route defines the endpoint (routes/blogPosts.js)
router.post('/',
    authenticate,                  // 2. Check if user is logged in
    authorize('admin', 'editor'),  // 3. Check if user has permission
    validate(schemas.create),      // 4. Validate request body
    blogPostsController.create     // 5. Execute business logic
);

// 6. Controller handles the request (controllers/blogPosts.js)
async create(req, res) {
    const { title, content, slug } = req.body;

    // 7. Save to database via Supabase
    const { data, error } = await supabase
        .from('blog_posts')
        .insert({ title, content, slug })
        .select();

    // 8. Return response
    res.status(201).json({ success: true, data: data[0] });
}

Middleware Explained

authenticate

Checks if the request has a valid JWT token. Adds req.user with user info if valid.

// Requires login
router.get('/protected', authenticate, handler);

authorize(...roles)

Checks if the user has one of the allowed roles. Must be used after authenticate.

// Only admin and editor can access
router.post('/create', authenticate, authorize('admin', 'editor'), handler);

validate(schema)

Validates request body/query/params against a Joi schema. Returns 400 error if invalid.

// Validate request body
router.post('/create', validate(createSchema), handler);

optionalAuth

Like authenticate but doesn't fail if no token. Sets req.user if logged in.

// Works for both logged in and anonymous
router.get('/public', optionalAuth, handler);

Authentication System

The application uses JWT (JSON Web Tokens) for authentication. Here's how it works:

Login Flow

User submits credentials

User enters email and password on the login page.

Frontend calls API

POST request to /api/auth/login with credentials.

Backend validates with Supabase

Supabase Auth checks credentials and returns JWT tokens.

Tokens stored in browser

Access token and refresh token saved to localStorage.

Token included in requests

Axios interceptor adds Authorization: Bearer <token> header to all API calls.

Using Authentication in Code

// Frontend: Using the AuthContext
import { useAuth } from '../context/AuthContext';

function MyComponent() {

            <p>Welcome, {user.fullName}!</p>

Adding API Endpoints

Learn how to add new API endpoints to the backend:

Step-by-Step Guide

Create the Validator (Optional)

Define validation rules in backend/src/validators/

// backend/src/validators/myResource.js
const Joi = require('joi');

const schemas = {
    create: Joi.object({
        name: Joi.string().required().min(3).max(100),
        description: Joi.string().optional()
    }),
    update: Joi.object({
        name: Joi.string().min(3).max(100),
        description: Joi.string()
    })
};

module.exports = { schemas };
Create the Controller

Add business logic in backend/src/controllers/

// backend/src/controllers/myResource.js
const { supabase } = require('../config/supabase');
const { ApiError } = require('../utils/errors');

const myResourceController = {
    async getAll(req, res) {
        const { data, error } = await supabase
            .from('my_table')
            .select('*');

        if (error) throw new ApiError(500, error.message);

        res.json({ success: true, data });
    },

    async create(req, res) {
        const { data, error } = await supabase
            .from('my_table')
            .insert(req.body)
            .select();

        if (error) throw new ApiError(400, error.message);

        res.status(201).json({ success: true, data: data[0] });
    }
};

module.exports = myResourceController;
Create the Routes

Define endpoints in backend/src/routes/

// backend/src/routes/myResource.js
const express = require('express');
const router = express.Router();
const controller = require('../controllers/myResource');
const { authenticate, authorize } = require('../middleware/auth');
const { validate } = require('../middleware/validate');
const { schemas } = require('../validators/myResource');

// Public routes
router.get('/', controller.getAll);
router.get('/:id', controller.getById);

// Protected routes
router.post('/', authenticate, authorize('admin', 'editor'), validate(schemas.create), controller.create);
router.put('/:id', authenticate, authorize('admin', 'editor'), validate(schemas.update), controller.update);
router.delete('/:id', authenticate, authorize('admin'), controller.delete);

module.exports = router;
Register the Routes

Add to backend/src/routes/index.js

// In backend/src/routes/index.js
const myResourceRoutes = require('./myResource');

// Add this line with other routes
router.use('/my-resources', myResourceRoutes);

Security Features

The application implements several security measures:

Backend Security

  • Helmet - Sets secure HTTP headers
  • CORS - Controls which origins can access API
  • Rate Limiting - Prevents brute force attacks
  • JWT Tokens - Secure authentication
  • Input Validation - Joi schema validation
  • RLS Policies - Database-level access control

Frontend Security

  • Protected Routes - Role-based access
  • Token Storage - Secure localStorage handling
  • Auto Logout - On token expiry
  • HTTPS - Encrypted communication
  • XSS Prevention - React's built-in escaping
Security Best Practices
  • Never commit .env files to version control
  • Always validate user input on both frontend and backend
  • Use HTTPS in production
  • Keep dependencies updated

Best Practices

Code Organization

  • Use meaningful names for files, functions, and variables
  • Group related files together
  • Write comments for complex logic

React Best Practices

  • Use functional components with hooks
  • Keep state as close as possible to where it's used
  • Avoid prop drilling - use context for deeply nested data
  • Memoize expensive calculations with useMemo
  • Handle loading and error states gracefully

API Best Practices

  • Always validate input data
  • Return consistent response formats
  • Use appropriate HTTP status codes
  • Implement pagination for large datasets
  • Log errors for debugging
